# email-to-telegram-bot

Маленький Telegram-бот, который уведомляет о новых письмах. Опрашивает почтовый
ящик по IMAP, берёт непрочитанные письма (опционально — только от заданного
отправителя), пересылает тему и тело в Telegram и помечает их прочитанными.

По умолчанию настроен на Яндекс-почту, но подойдёт любой IMAP-сервер.

## Конфигурация

Все параметры — в `.env` (см. [`.env.example`](.env.example)). Секретов в коде нет.

| Переменная | Назначение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | куда слать письма (id чата/пользователя) |
| `YANDEX_EMAIL` | логин почтового ящика |
| `YANDEX_PASSWORD` | пароль приложения (не основной пароль аккаунта) |
| `TARGET_EMAIL` | фильтр по отправителю; пусто = принимать от всех |
| `IMAP_HOST` | IMAP-сервер (по умолчанию `imap.yandex.ru`) |
| `POLL_INTERVAL` | период опроса в секундах (по умолчанию 60) |
| `TELEGRAM_PROXY` | необязательно — прокси для Telegram (см. ниже) |

## Запуск

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить своими значениями
python bot.py
```

Или через Docker:

```bash
cp .env.example .env   # заполнить
docker compose up -d --build
docker compose logs -f
```

## Если Telegram недоступен напрямую

Если из вашей сети недоступен `api.telegram.org` — запускайте бота за HTTP/SOCKS-прокси
(`TELEGRAM_PROXY=http://HOST:PORT` или `socks5://HOST:PORT`) либо внутри сетевого
пространства VPN.

## Как это работает

- Раз в `POLL_INTERVAL` секунд бот логинится в IMAP и ищет непрочитанные письма
  (если задан `TARGET_EMAIL` — только от этого отправителя).
- Обрабатываются **все** подходящие письма, а не только последнее.
- Письмо помечается прочитанным **только после** успешной отправки в Telegram —
  при сбое отправки оно повторится в следующем цикле.
- Заголовки и тело декодируются по charset, длинные письма обрезаются.
