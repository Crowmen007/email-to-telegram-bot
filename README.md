# email-to-telegram-bot

Маленький Telegram-бот, который уведомляет о новых письмах. Опрашивает почтовый
ящик по IMAP, берёт непрочитанные письма (опционально — только от заданного
отправителя), отправляет в Telegram и помечает их прочитанными.

HTML-письма по умолчанию **рендерятся в картинку** (вид как в почте) и шлются
фото; письма без HTML — текстом с лёгкой разметкой. Рендер можно отключить
(`RENDER_HTML=0`), тогда всё уходит текстом.

По умолчанию настроен на Яндекс-почту, но подойдёт любой IMAP-сервер.

## Конфигурация

Все параметры — в `.env` (см. [`.env.example`](.env.example)). Секретов в коде нет.

| Переменная | Назначение |
|---|---|
| `TELEGRAM_BOT_TOKEN` | токен бота от @BotFather |
| `TELEGRAM_CHAT_ID` | куда слать письма (id чата/пользователя) |
| `YANDEX_EMAIL` | логин почтового ящика |
| `YANDEX_PASSWORD` | пароль приложения (не основной пароль аккаунта) |
| `TARGET_EMAIL` | фильтр по отправителю; пусто = принимать от всех |
| `IMAP_HOST` | IMAP-сервер (по умолчанию `imap.yandex.ru`) |
| `POLL_INTERVAL` | период опроса в секундах (по умолчанию 60) |
| `TELEGRAM_PROXY` | необязательно — прокси для Telegram (см. ниже) |
| `RENDER_HTML` | `1` (по умолчанию) — HTML-письма в картинку; `0` — слать текстом |
| `RENDER_WIDTH` | ширина рендера в px (по умолчанию 600) |

## Запуск

```bash
python3 -m venv .venv && . .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env   # заполнить своими значениями
python bot.py
```

Или через Docker:

```bash
cp .env.example .env   # заполнить
docker compose up -d --build
docker compose logs -f
```

## Если Telegram недоступен напрямую

Если из вашей сети недоступен `api.telegram.org` — запускайте бота за HTTP/SOCKS-прокси
(`TELEGRAM_PROXY=http://HOST:PORT` или `socks5://HOST:PORT`) либо внутри сетевого
пространства VPN.

## Как это работает

- Раз в `POLL_INTERVAL` секунд бот логинится в IMAP и берёт непрочитанные письма
  (`UNSEEN`). Если задан `TARGET_EMAIL`, лишние отсеиваются по отправителю уже
  **на стороне бота** (по адресу из заголовка `From`), а не серверным поиском.
- Почему не `IMAP SEARCH FROM`: у ряда провайдеров (в т.ч. Яндекс) серверный
  поиск `FROM` не матчит полный адрес `user@domain.tld` и молча возвращает ноль
  писем — поэтому на сервере запрашиваются только `UNSEEN`, а отправитель
  сверяется в коде.
- Обрабатываются **все** подходящие письма, а не только последнее.
- Письмо помечается прочитанным **только после** успешной отправки в Telegram —
  при сбое отправки оно повторится в следующем цикле.
- HTML-письмо рендерится в PNG через `wkhtmltoimage` и шлётся фото; если HTML
  нет или рендер не удался — уходит текстом (заголовки/тело по charset, лёгкая
  разметка). В Docker-образе `wkhtmltoimage` уже установлен; для запуска без
  Docker поставьте его в систему или отключите рендер (`RENDER_HTML=0`).
